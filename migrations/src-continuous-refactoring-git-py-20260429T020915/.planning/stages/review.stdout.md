1. `phase-2-reorganize-domain-flow.md` hard-codes a private implementation shape in its Definition of Done: "`discard_workspace_changes()` and `revert_to()` both delegate their reset and clean work through `_reset_hard_and_clean()`". That is more specific than the plan’s stated taste or behavioral goal, and it weakens Phase 4’s “trim helper shapes” freedom by turning one helper name into a required artifact. Better: require one shared reset/clean path with unchanged behavior, without naming the helper.

2. `phase-4-trim-helper-shapes-and-verify-surface.md` has a fuzzy start gate: “the public symbol set ... still matches the surface this migration is preserving.” That is directionally right, but not concrete enough for phase gating because “the surface this migration is preserving” lives in human intent, not a named artifact. Tie the precondition to something checkable, e.g. the Phase 1/4 export coverage or the `git.py.__all__` symbol set pinned at migration start.
